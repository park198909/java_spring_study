자바 문제만들기

1. 생성자를 정의하는 이유를 작성하시오
	객체를 생성해서 변수와 메서드를 사용하기 위해
2. 따로 생성자를 정의하지 않아도 컴파일러가 자동으로 생성하는 생성자의 이름은?
	기본 생성자 == 디폴트 생성자
3. getter() , setter() 함수를 생성하여 사용하는 이유는?
	변수를 통제하기 위해서, 정보보안
4. 다음 빈칸을 채우시오
	1. (public) - 외부패키지 및 내부패키지 접근가능
	2. (default) - 내부패키지 접근가능, 외부패키지 접근불가
	3. (protected) - 내부패키지 접근가능, 상속을 통해 외부패키지 접근가능
	4. (private) - 클래스 내부에서만 접근가능
	
5. A클래스가 B클래스를 상속하고 있다.  이때, B클래스의 this가 가르키는 것은?
	1. A클래스의 메서드()
	2. B클래스의 메서드()
	3. A클래스의 인스턴스 주소값
->	4. B클래스의 인스턴스 주소값
	5. B클래스의 상수

6. 다음 중 정적 메모리영역에 있지 않은 것은?
	1. final 상수
	2. static 변수
	3. static 메서드()
->	4. private 메서드()
	5. 리터럴상수
	
7. 싱글톤 패턴을 사용하는 가장 큰 이유는?
	객체를 하나만 생성해서 사용하므로 자원이 아껴진다
8. 상위클래스와 동일한 메서드를 재정의하여 사용하는 것을 의미하는 단어는?
	메서드 오버라이딩
9. 상속관계 : C <- B <- A  일 때, C에서 A에 있는 메서드를 사용 가능할까?
	예
10. 상속관계 : C <- B <- A  일 때, A a = new C(); 가 가능할까? 이유는?
	가능, C가 A의 하위클래스이므로 A를 포함한 객체이다
